[72. 编辑距离 - 力扣（LeetCode）](https://leetcode.cn/problems/edit-distance/description/)
### 核心口诀（背诵这个就够了）

> **“相等直接抄，不等三选一；越界加一返回，算过直接拿。”**

---

### 代码解剖与逻辑映射

我们把核心函数 `dfs(i, j)` 翻译成大白话：它的意思是“把字符串 `s` 的前 `i` 个字符，变成字符串 `t` 的前 `j` 个字符，最少需要几步？”

#### 1. 越界加一返回 (边界条件)

当指针跑到 `-1` 的时候，说明有一个字符串已经空了。

- `if (i < 0) return j + 1;` -> `s` 变空了，但 `t` 还有 `j+1` 个字符，我只能疯狂**插入** `j+1` 次。
    
- `if (j < 0) return i + 1;` -> `t` 变空了，但 `s` 还有 `i+1` 个字符，我只能疯狂**删除** `i+1` 次。
    

#### 2. 算过直接拿 (备忘录优化)

- `if (memo[i][j] != -1) return memo[i][j];` -> 这道题有大量重复子问题，如果这个状态之前算过了，直接返回，这就是 `memo` 数组唯一的用处。
    

#### 3. 相等直接抄 (状态转移 1)

- `if (s[i] == t[j]) return memo[i][j] = dfs(i - 1, j - 1);`
    
- **解释：** 如果当前两个字母一模一样（比如都是 'a'），太棒了，这步不需要任何操作。问题直接缩小规模，去比较 `i-1` 和 `j-1`。
    

#### 4. 不等三选一 (状态转移 2 - 最核心部分)

如果 `s[i]` 和 `t[j]` 不一样，你有三种修改手段，每次手段算作 1 步操作，你要选这三种里**最小**的那个：

- **手段 A：删除** -> `dfs(i - 1, j)`
    
    - 把 `s[i]` 删掉。既然删了，就用 `s[i-1]` 去和 `t[j]` 继续比。
        
- **手段 B：插入** -> `dfs(i, j - 1)`
    
    - 在 `s[i]` 后面强行插入一个和 `t[j]` 一样的字符。既然末尾匹配了，那就用 `s[i]` 去和 `t[j-1]` 继续比。
        
- **手段 C：替换** -> `dfs(i - 1, j - 1)`
    
    - 把 `s[i]` 强行替换成 `t[j]`。既然替换后两边一样了，双双抵消，用 `s[i-1]` 去和 `t[j-1]` 比。
        
- **代码体现：** `Math.min(Math.min(删除, 插入), 替换) + 1步操作`

### 结构化记忆模板 (闭上眼睛在脑海里默写框架)

Java

```
// 1. 全局变量准备：拆成字符数组 s 和 t，准备 memo 数组全部填满 -1
class Solution {
    // ...主函数负责初始化和调用 dfs(s.length - 1, t.length - 1)...

    int dfs(int i, int j) {
        // 步骤1：边界检查 (越界)
        if (i < 0) return j + 1;
        if (j < 0) return i + 1;

        // 步骤2：查备忘录
        if (memo 查到了) return memo[i][j];

        // 步骤3：核心逻辑分支
        if (当前字符相等) {
            记录并返回 dfs(i-1, j-1);
        } else {
            记录并返回 min(删除, 插入, 替换) + 1;
        }
    }
}
```
