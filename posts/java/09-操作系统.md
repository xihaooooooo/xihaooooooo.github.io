# 操作系统

---

## 1. 进程 vs 线程

| | 进程 | 线程 |
|--|------|------|
| 定义 | 资源分配的基本单位 | CPU 调度的基本单位 |
| 资源 | 独立地址空间、文件描述符 | 共享进程的内存、文件 |
| 通信 | IPC（管道、共享内存、消息队列、Socket） | 直接读写共享变量 |
| 开销 | 创建/切换/销毁大 | 小 |
| 崩溃 | 不会影响其他进程 | 一个线程崩了可能导致整个进程退出 |

## 2. 死锁

### 四个必要条件

1. **互斥**：资源只能被一个线程持有
2. **持有并等待**：持有资源并等待获取其他资源
3. **不可剥夺**：持有的资源不能被强制拿走
4. **循环等待**：若干线程形成等待环

### 解决方案

| 方法 | 说明 |
|------|------|
| 资源有序分配 | 所有线程按同一个顺序申请锁 |
| 一次性申请 | 申请所有资源时加锁，不满足就放弃 |
| 超时释放 | tryLock(timeout) 超时回退 |
| 死锁检测 | 周期检测资源分配图，手动处理 |

## 3. 进程调度算法

| 算法 | 说明 |
|------|------|
| FCFS（先来先服务） | 非抢占，长作业有利 |
| SJF（最短作业优先） | 平均等待最短，需预测作业长度 |
| 优先级调度 | 可能饥饿 → 老化（随时间提高优先级） |
| 时间片轮转（RR） | 公平，响应快 |
| 多级反馈队列 | 不同优先级队列 + 不同时间片 + 抢占（最复杂也最实用） |

## 4. 进程间通信（IPC）

| 方式 | 特点 |
|------|------|
| 管道（Pipe） | 父子进程间单向通信，数据在内存 |
| 命名管道（FIFO） | 任意进程间 |
| 共享内存 | 最快（直接读内存），需配合信号量 |
| 消息队列 | 异步，按消息读取，不阻塞 |
| 信号量（Semaphore） | 控制多进程访问共享资源（PV 操作） |
| Socket | 跨网络通信 |

## 5. 虚拟内存

- 每个进程拥有独立的虚拟地址空间（32 位 = 4GB，64 位 = 超大）
- 内存分页，OS 维护**页表**完成虚拟地址 → 物理地址映射
- **缺页中断**：访问的页不在物理内存 → 从磁盘加载
- **页面置换算法**：

| 算法 | 说明 |
|------|------|
| OPT | 最优置换（理论最优，无法实现） |
| LRU | 最近最少使用 |
| LFU | 最不频繁使用 |
| Clock | 近似 LRU（访问位 + 修改位） |

## 6. 零拷贝

### 传统 IO（4 次拷贝，4 次上下文切换）

```
磁盘 → 内核缓冲区 → 用户缓冲区 → Socket 缓冲区 → 网卡
```

### mmap + write（3 次拷贝）

- mmap 直接映射内核缓冲区到用户空间 → 减少一次拷贝
- RocketMQ 用 mmap

### sendfile（2 次拷贝，2 次上下文切换）

```
磁盘 → 内核缓冲区 → Socket 缓冲区（SG-DMA）→ 网卡
```

- Kafka、Nginx 用 sendfile

## 7. 用户态 vs 内核态

- **用户态**：受限权限，不能直接访问硬件/内核数据
- **内核态**：最高权限，可执行任何 CPU 指令
- **切换方式**：系统调用、中断、异常
- 切换开销：保存上下文、更新寄存器、刷新 TLB

## 8. select / poll / epoll

| | select | poll | epoll |
|--|--------|------|-------|
| 数据结构 | 位图（fd_set） | 链表 | 红黑树 + 就绪链表 |
| 最大连接数 | 1024（可编译修改） | 无限制 | 无限制 |
| 工作模式 | 轮询 O(n) | 轮询 O(n) | 回调 O(1) |
| 内存拷贝 | 每次全量拷贝 | 每次全量拷贝 | mmap 映射，共享内存 |
| 触发方式 | 水平触发 | 水平触发 | 水平/边缘触发（ET） |

> epoll 是 Linux 下高性能网络编程基石。Nginx、Redis、Netty 都基于 epoll。

## 9. 线程同步方式

| 方式 | 说明 |
|------|------|
| 互斥锁（Mutex） | 排他访问 |
| 读写锁 | 读共享、写排他 |
| 自旋锁 | 不挂起，循环等待（适合临界区短） |
| 信号量（Semaphore） | 控制并发数量（>=0 整数） |
| 条件变量 | wait / signal 模式 |
| 屏障（Barrier） | 等待所有线程到达同一位置 |

## 10. 内存管理

### 连续分配

- 首次适配 / 最佳适配 / 最坏适配
- 外部碎片 → 紧凑

### 非连续分配

- **分页**：内存分成固定大小页框 + 页表映射 → 内部碎片
- **分段**：按逻辑单元（代码段、数据段）→ 外部碎片
- **段页式**：先分段再分页，取两者之长
