# Java 集合框架

---

## 1. ArrayList vs LinkedList

| | ArrayList | LinkedList |
|--|-----------|------------|
| 底层结构 | `Object[]` 数组 | 双向链表 |
| 随机访问 | O(1) | O(n) |
| 头/尾增删 | O(n) / O(1) | O(1) |
| 内存占用 | 连续空间 | 节点额外存储 prev/next 指针 |
| 默认容量 | 10，扩容 1.5 倍 | — |

## 2. HashMap 原理（超高频）

### 数据结构演变

- **JDK 1.7**：数组 + 链表（头插法，多线程扩容时可能形成循环链表死循环）
- **JDK 1.8**：数组 + 链表/红黑树（链表长度 ≥ 8 且数组长度 ≥ 64 → 转红黑树；≤ 6 退化回链表）

### put 流程

```
1. 计算 key 的 hashCode() → h ^ (h >>> 16) 扰动
2. (n - 1) & hash 定位桶下标
3. 桶为空 → 直接插入
4. 桶非空 → 遍历链表/红黑树
   - key 相同 → 覆盖 value
   - 尾部插入 → 链表长 ≥ 8 → 树化
5. size > threshold → resize() 扩容
```

### 扩容机制

- 默认初始容量 **16**，负载因子 **0.75**
- 扩容为原来的 **2 倍**
- 为什么容量是 2 的幂？`(n-1) & hash` 等价于 `hash % n`（取模优化），扩容 rehash 时节点只可能在 **原位置** 或 **原位置 + 旧容量**

### HashMap 与 Hashtable

| | HashMap | Hashtable |
|--|---------|-----------|
| 线程安全 | ❌ | ✅ synchronized 锁全表 |
| null | 允许 key/value 为 null | ❌ 不允许 |
| 效率 | 高 | 低（几乎淘汰） |

## 3. ConcurrentHashMap

### 1.7 vs 1.8

| | JDK 1.7 | JDK 1.8 |
|--|---------|---------|
| 结构 | Segment[] + HashEntry[] | Node[] + 链表/红黑树 |
| 锁 | 分段锁（Segment 继承 ReentrantLock） | CAS + synchronized 锁头节点 |
| 锁粒度 | 16 个 Segment（并发度 16） | 桶级别 |
| 实现 | 先 Hash → 定位 Segment → 再 Hash → 定位 Entry | 一次 Hash → 定位 Node |

---

## 4. HashSet / TreeSet

- **HashSet**：基于 HashMap（value 固定为一个 Object 常量）
- **TreeSet**：基于 TreeMap（红黑树），可排序
- **LinkedHashSet**：继承 HashSet，底层 LinkedHashMap（双向链表维护插入顺序）

---

## 5. TreeMap

- 底层：**红黑树**
- key 必须可比较（实现 Comparable 或传入 Comparator）
- put/get/remove：O(log n)

---

## 6. fail-fast 与 fail-safe

| | fail-fast | fail-safe |
|--|-----------|-----------|
| 机制 | 迭代过程中结构修改 → 抛 ConcurrentModificationException | 迭代副本 |
| 检测方式 | modCount != expectedModCount | — |
| 代表 | ArrayList, HashMap | CopyOnWriteArrayList, ConcurrentHashMap |
| 优缺点 | 快速失败 | 数据可能过时但不会炸 |

---

## 7. ConcurrentHashMap 读不加锁

- Node 的 val 和 next 都是 `volatile` → 保证可见性
- 数组通过 `Unsafe.getObjectVolatile` 读取 → 总是读到最新

---

## 8. Collections 工具类

- `Collections.synchronizedList()`：返回线程安全的包装 List（锁所有方法）
- `Collections.unmodifiableList()`：返回不可变 List
- `Collections.sort()`：底层 Arrays.sort()，元素量小时用快排，大时用 TimSort
