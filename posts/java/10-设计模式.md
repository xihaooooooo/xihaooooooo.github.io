# 设计模式

---

## 高频必背

### 1. 单例模式

**双重检查锁定（DCL）**

```java
public class Singleton {
    // volatile 防止指令重排（分配内存 → 初始化 → 赋值引用 可能乱序）
    private static volatile Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                     // 第一次检查，避免不必要的同步
            synchronized (Singleton.class) {
                if (instance == null) {             // 第二次检查，确保只创建一次
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

其他实现：**静态内部类**（推荐，懒加载 + 线程安全）、**枚举**（最安全，防反射破坏）

---

### 2. 工厂模式

- **简单工厂**：一个工厂类根据参数创建不同产品（违反开闭原则）
- **工厂方法**：定义一个创建对象的接口，子类决定实例化哪个类
- **抽象工厂**：创建**相关对象家族**（如不同操作系统的 UI 组件族）

```java
// 工厂方法
interface ProductFactory {
    Product create();
}
class ConcreteFactoryA implements ProductFactory {
    public Product create() { return new ProductA(); }
}
```

---

### 3. 代理模式

为其他对象提供一种代理以**控制访问**，核心是三要素：**接口 → 真实对象 → 代理对象**。

| 类型 | 实现 |
|------|------|
| 静态代理 | 编译期生成代理类 |
| JDK 动态代理 | 运行时，基于接口（InvocationHandler） |
| CGLIB | 运行时，基于子类继承 |

应用：Spring AOP、MyBatis Mapper、RPC 远程调用

---

### 4. 观察者模式

定义一对多依赖，当一个对象状态改变，所有依赖者自动收到通知。

```
Subject（主题）← 持有 Observer 列表
  notify() → 遍历 Observer.update()
```

应用：Spring 事件机制、消息队列发布订阅、GUI 事件监听

---

### 5. 策略模式

定义一系列算法，封装起来让它们可互相替换，避免 if-else 分支。

```java
interface Strategy { void execute(); }
class StrategyA implements Strategy { ... }
class StrategyB implements Strategy { ... }

// Context 持有 Strategy，根据情况切换
```

应用：Comparator 排序、支付方式选择

---

### 6. 装饰器模式

动态地给对象添加额外职责，比继承更灵活。

```
InputStream is = new BufferedInputStream(new FileInputStream("a.txt"));
// BufferedInputStream 装饰 FileInputStream，增强了缓冲功能
```

---

### 7. 责任链模式

多个对象都有机会处理请求，将其连成链，沿链传递直到被处理。

```
Handler A → Handler B → Handler C
  ↓ 处理          处理          处理
```

应用：Servlet Filter、Spring Interceptor、审批流程

---

### 8. 模板方法模式

定义算法骨架，将可变步骤延迟到子类。

```java
abstract class AbstractClass {
    public final void templateMethod() {  // final 防止子类篡改骨架
        step1();
        step2();  // 抽象方法，子类实现
        step3();
    }
    protected abstract void step2();
}
```

应用：JdbcTemplate、Spring Bean 初始化流程

## 其他常见模式

| 模式 | 说明 | 应用 |
|------|------|------|
| **适配器** | 转换接口（不兼容→兼容） | HandlerAdapter、InputStreamReader |
| **建造者** | 分步构建复杂对象 | StringBuilder、Lombok @Builder |
| **原型** | 克隆已有对象 | Object.clone()、Spring prototype |
| **享元** | 共享细粒度对象 | Integer 缓存池(-128~127)、String 常量池 |
| **外观** | 统一入口，屏蔽子系统 | SLF4J 日志门面 |
| **状态** | 状态驱动行为 | 订单状态流转 |

## 速记分类

| 分类 | 模式 |
|------|------|
| 创建型 (5) | 单例、工厂方法、抽象工厂、建造者、原型 |
| 结构型 (7) | 代理、适配器、装饰器、外观、桥接、组合、享元 |
| 行为型 (11) | 观察者、策略、模板方法、责任链、状态、迭代器、命令、备忘录、中介者、解释器、访问者 |

> 面试中最常问：**单例（手写 DCL）、代理（JDK vs CGLIB）、工厂方法、观察者、策略**
