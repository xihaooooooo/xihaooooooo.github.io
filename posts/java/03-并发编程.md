# Java 并发编程

---

## 1. 线程生命周期

```
NEW → RUNNABLE → BLOCKED → WAITING → TIMED_WAITING → TERMINATED
```

- `BLOCKED`：等待获取 synchronized 锁
- `WAITING`：wait() / join() / LockSupport.park()
- `TIMED_WAITING`：sleep(time) / wait(timeout) / join(timeout)

---

## 2. synchronized 锁升级（JDK 1.6+）

**无锁 → 偏向锁 → 轻量级锁 → 重量级锁**（单向不可降级）

| 锁状态 | 场景 | 实现 |
|--------|------|------|
| 偏向锁 | 无竞争、单线程反复获取 | Mark Word 记录线程 ID |
| 轻量级锁 | 竞争但交替执行 | CAS 自旋 |
| 重量级锁 | 激烈竞争 | 系统互斥量（挂起线程） |

---

## 3. volatile

- **可见性**：修改立即可见（强制从主内存读写，CPU 缓存失效）
- **禁止指令重排**：内存屏障（Memory Barrier）
- **不保证原子性**：`i++` 操作不安全（读→改→写三段）
- 使用场景：状态标记（double-check locking 中的 flag）、DCL 单例

---

## 4. CAS（Compare And Swap）

```
while (!compareAndSwap(内存地址, 期望值, 新值)) {
    // 自旋重试
}
```

- 底层：CPU 原子指令 `lock cmpxchg`
- **ABA 问题**：解决 → `AtomicStampedReference`（加版本号）
- **自旋开销**：长时间竞争浪费 CPU
- Java 并发包基石：`AtomicInteger`、`AtomicReference` 等

---

## 5. AQS（AbstractQueuedSynchronizer）

核心三要素：**state（volatile int）+ CLH 队列 + CAS**

- **独占模式**：ReentrantLock（state 记录重入次数）
- **共享模式**：Semaphore（state 记录剩余许可）、CountDownLatch
- `tryAcquire` / `tryRelease` → 子类重写，AQS 负责排队/唤醒

---

## 6. ReentrantLock vs synchronized

| | synchronized | ReentrantLock |
|--|-------------|---------------|
| 实现 | JVM 层面（monitor） | JDK 层面（AQS） |
| 锁释放 | 自动（代码块结束/异常） | 手动 unlock（finally 中） |
| 可中断 | ❌ | ✅ lockInterruptibly() |
| 可超时 | ❌ | ✅ tryLock(timeout) |
| 公平锁 | ❌ 非公平 | ✅ 可选公平/非公平 |
| 多条件 | ❌ | ✅ 多个 Condition |
| 性能 | 1.6 后近似 | 近似 |

---

## 7. ThreadLocal

- 每个 Thread 中有 `ThreadLocalMap`，key 是弱引用的 ThreadLocal 实例
- **内存泄漏原因**：key 被 GC 回收，但 value 是强引用 → Entry 无法被回收
- **解决**：使用完必须调用 `remove()`
- `InheritableThreadLocal` 可传递给子线程

## 8. 线程池（ThreadPoolExecutor）

### 7 大参数

```
corePoolSize   核心线程数
maxPoolSize    最大线程数
keepAliveTime  非核心线程空闲存活时间
unit           时间单位
workQueue      阻塞队列
threadFactory  线程工厂
handler        拒绝策略
```

### 工作流程

```
提交任务
  ↓
核心线程数 < corePoolSize？ → 是 → 创建核心线程
  ↓ 否
队列未满？ → 是 → 入队等待
  ↓ 否
线程数 < maxPoolSize？ → 是 → 创建非核心线程
  ↓ 否
执行拒绝策略
```

### 4 种拒绝策略

| 策略 | 行为 |
|------|------|
| AbortPolicy | 抛 RejectedExecutionException（默认） |
| CallerRunsPolicy | 由提交任务的线程自己执行 |
| DiscardOldestPolicy | 丢弃队列最旧的任务 |
| DiscardPolicy | 静默丢弃 |

### 常用线程池的坑

| 线程池 | 风险 |
|--------|------|
| FixedThreadPool | LinkedBlockingQueue 无界 → OOM |
| CachedThreadPool | SynchronousQueue + maxPoolSize=Integer.MAX_VALUE → CPU 爆炸 |
| SingleThreadExecutor | LinkedBlockingQueue 无界 → OOM |

> **阿里巴巴规范**：强制用 ThreadPoolExecutor 手动创建，不允许用 Executors

---

## 9. 创建线程的方式

1. 继承 `Thread` 类
2. 实现 `Runnable` 接口
3. 实现 `Callable<T>` + `FutureTask`（有返回值和异常）
4. 线程池

推荐方式：线程池（复用、管理），Callable（需要返回值时）

---

## 10. wait / notify vs LockSupport

- `wait()` / `notify()` 必须在 synchronized 块内调用
- `LockSupport.park()` / `unpark()` 无此限制，且 unpark 可先于 park 调用（许可证机制）

---

## 11. JUC 常用工具类

| 类 | 作用 |
|----|------|
| CountDownLatch | 等 N 个线程完成，不可复用 |
| CyclicBarrier | N 个线程相互等待到齐，可复用 |
| Semaphore | 限制同时访问资源的线程数（限流） |
| Exchanger | 两个线程交换数据 |
| CompletableFuture | 异步编排（thenApply/thenCombine/exceptionally） |
